FIXED & FINAL

🔧 Bugs Fixed

BUG Model → DeviceModel rename. The Model class name clashes directly with Laravel's Illuminate\Database\Eloquent\Model base class. Renamed the Eloquent class to DeviceModel and the DB table to device_models. The FK column is now device_model_id everywhere.
BUG Double JSON encoding in DeviceLogService. DeviceLog already has 'details' => 'array' in $casts. Calling json_encode() manually before saving produced double-encoded JSON strings. Fixed by passing a plain PHP array directly to DeviceLog::create().
BUG Stray } in DeviceOwnerSwapController. An extra closing brace at the end of the file caused a PHP parse error and prevented the entire app from booting. Removed.
BUG DeviceMovement model $fillable mismatch. from_department_id and to_department_id were listed in $fillable but those columns don't exist in the migration. Removed from $fillable; they would have caused silent insert failures.
BUG DeviceLog migration column type. The details column was declared as text() but the model casts it as array (JSON). Changed to json() for proper DB-level type alignment.
DESIGN Removed redundant site_id from devices table. Site is already reachable via device → location → site. Keeping a second site_id on devices creates a risk of data inconsistency.
DESIGN Simplified StatusLabel. The separate StatusLabel enum table added no benefit over a name string column directly on statuses. Merged into one statuses table with a unique name field.
DESIGN Added department_id to devices table. The class diagram and controllers referenced it but it was missing from the original devices migration.
